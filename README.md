# Data Center Intelligence Suite

Tracks **data-center value-chain developments and partnerships across India + the GCC**
(Compute / Cooling / Power / Network / Colo / Build), split by *who is speaking*:

| Tab | Source type | Status |
|-----|-------------|--------|
| **SS1 News** | journalists talk about them (RSS firehose) | ✅ live |
| **SS2 Policy** | governments rule on them (think-tanks + India/GCC gov feeds) | ✅ live |
| **SS3 Disclosure** | they talk about themselves (SEC EDGAR full-text) | ✅ live |
| **SS4 OSINT** | footprints (Reddit + Greenhouse/Lever/Adzuna jobs) | ✅ live (add ATS tokens / Adzuna key for volume) |
| **SS5 Ranked** | scored roll-up over SS1–SS4 (BD Framework v3 weights) | ✅ live |

All five worksheets live in **one** Google Sheet ("Datacentre"). Config-driven,
runs entirely on GitHub Actions cron — no servers.

Two scheduled jobs: `dc-news.yml` (SS1, ~every 3h) and `dc-pipeline.yml`
(SS2–SS5, daily).


## Adding a source (the whole workflow)

1. Open `dc_config.py` — the single source registry.
2. Append one line to the right dict/list (a feed URL, an ATS token, a query).
3. Run `python check_dc_sources.py` — confirms every source is live before it ships.

That's it. Dead/blocked sources are kept as commented stubs with the reason.

## Files

| File | Role |

## Secrets (GitHub → Settings → Secrets)

Public repo — **never** put credentials in code. Set as Actions secrets:

- `GOOGLE_SERVICE_ACCOUNT_JSON` — service-account JSON (share the Datacentre sheet with its email, Editor)
- `DC_SPREADSHEET_ID` — the Datacentre sheet id
- `ADZUNA_APP_ID`, `ADZUNA_APP_KEY` — for SS4 Adzuna (when wired)
- `SEC_USER_AGENT` — "Name email@example.com" for EDGAR (when wired)

## AI boundary (locked — Phase 4 PRD, 2026-07-04)

Deterministic rules + curated lists (`dc_config.py` registries) own **role, type,
and priority**. AI (free OpenRouter Nemotron, cached, non-fatal) may only:

1. **Drop or label evidence** — the SS3 signal judge is an *extraction filter*:
   every quote it returns is substring-verified against the source passage; it
   can never add facts or promote a company.
2. **Downgrade** — flag ambiguous rows as Noise/Market-signal. Never upgrade a
   company into Prospect/P1.
3. **Write grounded prose** — always prefixed `🤖AI-draft:` and grounded on
   sheet data only.

**Entity Overrides is the only human upgrade path.**

## Branching & CI cron

The daily crons commit caches/state (`*_cache.json`, `dc_news_state.json`,
`dc_trend.json`, `dashboard/data.json`) to `main` with `[skip ci]`. Feature
branches must **never** modify those files (cache schema changes are handled
read-tolerantly in code), so rebase-before-merge stays clean. If a cache ever
conflicts: take main's copy (`git checkout origin/main -- <file>`) — caches
are derived, latest run wins. Work lands via `feat/m<N>-<slug>` branches → PR
→ merge to `main`.

## Product surfaces

- **Dashboard (Vercel)** — the product: `dashboard/` static site fed by

- **Google Sheet ("Datacentre")** — the data-audit surface: every evidence row,

## Architecture (Phase 4 complete — two-layer model)

**Layer 1 — signal engine:** SS1 News · SS2 Policy (7-class classifier) ·
SS3 Disclosure (SEC full-text + per-CIK submissions + press releases + earnings
transcripts, all through one KWIC + verified-signal judge) · SS4 OSINT (jobs,
tenders, facilities, vendor footprints) → SS5 Ranked (tier-weighted Signal
Score + corroboration gate).

**Layer 2 — BD product:** deterministic role/type (`dc_classify`) → BD Priority
Score with visible factor breakdown + fee viability (`dc_bd`) → balanced
curated MD View (`dc_md`) → India state intelligence (`dc_states` over
`knowledge/india_state_bank.md`) → Vercel dashboard (`dc_export` →
`dashboard/data.json`).

**Registries own everything** (`dc_config.py` §11): signal taxonomy, company
types, source tiers, policy classes, BD factor weights, backer scale, state
mappings. **Add a sector later** = a new set of registries + a sibling
`data-<sector>.json` + one dashboard nav card — no engine changes.

**Deferred adapters** (admitted by docs/SOURCES_PRD.md, pending live
verification): PARIVESH + CCI page monitors, PIB page monitor, Wikidata alias
enrichment, Lever/Ashby boards.
